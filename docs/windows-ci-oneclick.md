# Windows one-click CI automation

**Status:** Windows CI prepared. Do not mark the app Windows-verified until the GitHub Actions `windows-latest` workflow or a real Windows test machine has run and passed.

## GitHub Actions workflow

Workflow: `.github/workflows/windows-oneclick.yml`

The workflow runs on `windows-latest` and verifies:

- Node 22 and pnpm availability.
- Dependency install with `pnpm install --frozen-lockfile`.
- PowerShell parser sanity for `scripts/windows-oneclick-check.ps1`.
- Core package builds, web build, desktop build, desktop tests, desktop release smoke.
- Pilot artifact tests, staged package generation, release verification, and manifest verification.
- PHI-free synthetic DATA generation with `pnpm strict-signoff:prepare`.
- Non-interactive staged package smoke through `DOUBLE-CLICK-AUTO-TEST.cmd --ci`.
- `pnpm microdent:oneclick:quick`.

The workflow uploads only PHI-safe reports/zips from `qa-runs/` and the Windows smoke output folder.

## Local Windows command

From a full repo checkout on Windows:

```powershell
pnpm install --frozen-lockfile
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\windows-oneclick-check.ps1 -Ci -UseSyntheticData -SkipInstall
```

For a fuller repo check, omit `-SkipInstall` and add `-Full`:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\windows-oneclick-check.ps1 -Full -UseSyntheticData
```

## Clinic package command

From the staged package root on Windows:

```cmd
DOUBLE-CLICK-AUTO-TEST.cmd
```

CI can run the same package path without prompts:

```cmd
DOUBLE-CLICK-AUTO-TEST.cmd --ci
```

## Synthetic data boundary

CI uses synthetic PHI-free DBF fixtures generated by `pnpm strict-signoff:prepare`. This proves the Windows package/build/service/preview path can run without real patient data.

Real copied clinic DATA validation remains a field-test step. Do not upload real DBF, SQLite, raw logs, screenshots, config files, or copied DATA folders to GitHub Actions artifacts.

## Return artifact

The expected safe artifact is:

```text
qa-runs\MicrodentModern-safe-results.zip
```

If the staged package was run manually, send back only the `MicrodentModern-safe-results.zip` opened by the smoke runner.
