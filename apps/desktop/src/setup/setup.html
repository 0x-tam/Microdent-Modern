Microdent desktop setup

Mac build machine vs Windows clinic PC: use this window on a Mac for local build/QA only. Clinic operators run the staged package on Windows — path examples below are for Windows targets, not Mac paths.
First-run flow (Windows-test readiness — tier 2):
  1. Enter DATA_ROOT, SQLITE_PATH, and optional BACKUP_DIR (Windows examples below)
  2. Save — config writes to %AppData%\Microdent\config.json on the target PC
  3. Run mirror import from the command line (CLI only — see package docs/)
  4. Open the app → Settings → Pilot checklist — confirm read-only Today / Patients / Schedule
  5. Sandbox write pilot (optional) — follow docs/PILOT-HANDOFF-PACK.md when IT approves
  6. Windows field execution (tier 3) — deferred until IT schedules a clinic PC run (docs/FIELD-TEST-START-HERE.md)

Point the app at your disposable Write-Sandbox copy (never the live production legacy DATA tree). Use absolute paths only. Write mode stays disabled until you change config manually for sandbox pilot work.

Production writes outside sandbox are not enabled in this pilot build. Read-only viewing is safe; sandbox commits require IT approval, disposable DATA, and explicit write-mode changes per docs/PILOT-HANDOFF-PACK.md.

Where files live: this app installs to one folder; config saves to %AppData%\Microdent\config.json; DATA_ROOT, mirror, and backups belong on clinic paths outside the install folder. In the staged package, start with docs/PILOT-HANDOFF-PACK.md (full tester index) or docs/PILOT-START-HERE.md. Location details: docs/windows-pilot-data-locations.md.

Windows: use drive-letter paths only in examples below — not hardcoded by the app. If a path contains spaces, quote it in manual PowerShell or CMD commands (for example "C:\ClinicData\My Sandbox\DATA").

UNC shares (\\server\share\…) are accepted but can be slow or unreliable for SQLite and DBF. Prefer a local drive letter when IT allows.

Check path: a folder name looks like production legacy. Use a disposable Write-Sandbox copy only (warn-only — you can still save if paths exist on disk).
Before you can save:
    Sandbox DATA folder with DBF files and the Write-Sandbox marker dotfile (see HANDOFF-README). Example: C:\ClinicData\Microdent\DATA
    SQLite mirror file for search and schedule. Example: C:\Users\Public\MicrodentModern\mirror\clinic.sqlite — outside the app install folder.
    Optional. Required before enabling sandbox writes on the bridge. Example: C:\Users\Public\MicrodentModern\backups on a local drive — mirror and backups must not live inside the app install folder (see windows-pilot-data-locations.md).
