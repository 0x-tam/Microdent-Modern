Microdent desktop setup

Point the app at your disposable Write-Sandbox copy (never live Microdent-Legacy). Use absolute paths only. Write mode stays disabled until you change config manually for sandbox pilot work.

Windows: use drive-letter paths only in examples below — not hardcoded by the app. If a path contains spaces, quote it in manual PowerShell or CMD commands (for example "C:\ClinicData\My Sandbox\DATA").

UNC shares (\\server\share\…) are accepted but can be slow or unreliable for SQLite and DBF. Prefer a local drive letter when IT allows.

Before you can save:
    Sandbox DATA folder with DBF files and .microdent-write-sandbox.json. Example: C:\ClinicData\Microdent\DATA
    SQLite mirror file for search and schedule. Example: C:\Users\Public\MicrodentModern\mirror\clinic.sqlite
    Optional. Required before enabling sandbox writes on the bridge. Example: C:\Users\Public\MicrodentModern\backups on a local drive (not inside the app install folder).
